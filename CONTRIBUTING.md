# 参与贡献（Contributing）

如果您希望为 Pulsar 项目贡献代码，请按照以下步骤操作：

1. **Fork** 本项目。
2. 创建您的功能分支：

   ```bash
   git checkout -b my-new-feature
   ```
3. 提交您的修改：

   ```bash
   git commit -am "Add some feature"
   ```
4. 将分支推送到您的仓库：

   ```bash
   git push origin my-new-feature
   ```
5. 创建一个新的 **Pull Request（PR）**。

---

## Pull Request（PR）提交规范

请在提交 PR 前遵循以下规则：

1. **遵循 Pulsar 的代码风格**。
2. **每个 PR 使用独立的分支**。
3. **每个 PR 仅包含一个功能或一个 Bug 修复**。
4. **每个 PR 尽量只包含一次提交（Single Commit）**。
5. **保持修改内容精简**，不要在提交中顺带重新格式化整个项目代码，否则会增加代码审查难度。
6. **仅包含代码格式调整的 PR（例如修改空格、TAB、换行符或代码风格）将不会被接受。** 如有相关建议，请通过 Issue 提出。
7. **单纯修正拼写错误（Typo）或代码重构（Refactoring）的 PR 将不会被接受。** 如果需要修改，请先提交一个标题以 **`TYPO`** 开头的 Issue 进行讨论。

---

## 总结

**代码越简洁、修改范围越明确，代码审查就越容易，您的 Pull Request 被接受的可能性也就越高。**
